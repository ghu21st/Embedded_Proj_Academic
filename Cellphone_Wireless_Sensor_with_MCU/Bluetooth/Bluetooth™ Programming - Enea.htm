The course is a three days hands-on introduction for those that need to know more about Bluetooth wireless technology. It contains a thorough walk-through of the Bluetooth wireless technology and pertinent standards. Half of the time is spent on programming applications that use the Bluetooth host stack implementation from Ericsson.
The development of the Bluetooth technology started in the beginning of the nineties. Today it is a well-defined standard and more than 2000 companies have joined as early developers, i.e. showing interest in developing products using Bluetooth. Initially the standard mobile market is targeted, e.g. mobile phones and palmtops. Later on will all aspects of the daily life be affected by Bluetooth enabled consumer products.

Objectives
To explain what Bluetooth is, including the standards, the architecture and also the application-programming interface supplied by Ericsson. After completing the course it should be possible to start to implement Bluetooth programming using the Ericsson host stack.

Who should attend?
Programmers and system designers and also to managers that wish to know more about Bluetooth.

Previous knowledge
Experience in the C-language is required. Knowledge in real-time programming and especially how it is applied in OSE is beneficial, but not required.

Course Material
Our original material included in the course is written in English.

 Course Contents 

Introduction to Bluetooth wireless technology
The concept, the devices and the area of usage.

Bluetooth wireless technology Architecture
What is the foundation for creating Bluetooth wireless technology enabled devices? How are services structured into layers and components? How do the components within the Bluetooth framework interact?

Radio and baseband constraints
The Bluetooth specifications regulates the topology, the links, the packet formats on the radio, the use cases for going from one state to another and the need for frequency hopping. The inherent security model and the radio are also explained.

API overview
A walkthrough of the API, given by the Bluetooth standard 1.1, as it appears to the programmer. What primitives exist, and how must these be used? What "states" are there? The description includes the upper layers of the stack, i.e. the host stack:
  -RFCOMM (serial cable emulation)
  -Service Discovery Protocol
  -The Data Base keeping track of the services
  -Telephony Control
  -Logical Link and Control Adaptation protocol (L2CAP)
  -Host Controller Interface and, finally
  -The physical transport using UART, RS232 or USB 1.0.
The approved profiles of Bluetooth are also discussed.

The Bluetooth host stack from Ericsson running on OSE
How does the Ericsson version of the stack differ from the standard? What is needed to get it up and running?


How to write a Bluetooth Application?
The focus of the course is on this section. There is a detailed sentation of all steps needed to make a running Bluetooth application on top of the Bluetooth host stack from Ericsson. E.g. how to create a physical link, obtain handles to physical links and logical links, the mysteries about Service Discovery Protocol and how to find other Bluetooth enabled  devices are revealed. The environment is the operating system OSE and the    services supplied with it.

Programming exercises
 A number of selected exercises are chosen to support the programmer with a good start for writing their own Bluetooth applications and to avoid the most common pitfalls. The platforms used are embedded targets together with cross-compilers, the operating system OSE from ENEA and the Bluetooth host stack from Ericsson. The development environment is a Windows NT workstation.

Tools
 What kind of tools may be beneficial for your development of your Bluetooth product?

New profiles
There are a number of new profiles that are close to approval that may be of interest in your coming products. What these do and how they accomplish it is explained.

Other Bluetooth stacks
There are competing stacks provided by other manufactories than Ericsson, e.g.from Mecel and Widcomm. Here are the differences for the BlueStack™ to the host stack from Ericsson examined.
